# 决策日志

## D001: 独立子项目 vs 扩展 backtest/
**决策**: 独立子项目 `grid-backtest/`
**原因**: 网格交易与波段/趋势策略是根本不同的范式（挂单 vs 择时），强行合并会导致配置/引擎层混乱。
**日期**: 2026-06-12

## D002: 等比网格 vs 等差网格
**决策**: 默认等比（geometric），可选等差（arithmetic）
**原因**: 股价变动本质是百分比驱动的，等比网格更符合金融市场特征。等差网格在低价区过于密集，高价区过于稀疏。
**日期**: 2026-06-12

## D003: 评分函数
**决策**: 默认 `calmar_sharpe`（0.5*Calmar + 0.3*Sharpe + 0.2*Ret）
**原因**: 需要同时平衡收益/风险/回撤，纯夏普可能漏选低回撤策略。
**日期**: 2026-06-12

## D004: 数据源策略
**决策**: efinance 主源 + akshare 自动回退
**原因**: 与现有 backtest/ 保持一致，efinance 速度更快，akshare 覆盖更全。
**日期**: 2026-06-12
