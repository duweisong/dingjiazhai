| 颜色 | 指标 | 含义 |
|------|------|------|

| 🔵 蓝条 | 夏普比率 | >1 良好，>2 优秀 |
| 🔴 红条 | 最大回撤 | 越低越好（-10% 优于 -25%） |
| 🟠 琥珀条 | 胜率 | >50% 即可，网格赚的是盈亏比 |

## 3. 命令行模式

### 基本用法

```bash
python main.py
```

默认参数：联创电子(002036)、240日周期、11档、0.5%步长、2020-2025

### 指定标的

```bash
python main.py --symbol 600519 --market SH     # 贵州茅台
python main.py --symbol 510300 --market SH     # 沪深300ETF
python main.py --symbol 300750 --market SZ     # 宁德时代
```

### 调整网格参数

```bash
python main.py --step 1.0 --levels 15 --period 480
```

### 参数扫描

自动遍历多组参数，输出最优组合：

```bash
python main.py --symbol 002036 --scan

# 自定义扫描范围
python main.py --symbol 002036 --scan \
    --scan-steps "0.5,1.0,1.5,2.0" \
    --scan-periods "120,240,480" \
    --scan-levels "7,11,15"
```

### 标的池扫描

```bash
python main.py --pool stock     # 10 只个股
python main.py --pool etf       # 6 只 ETF
```

### 生成图表

```bash
python main.py --plot           # 保存到 output/
python main.py --plot --show    # 弹窗显示
```

### 启用 MA 闸门

```bash
python main.py --gate --gate-period 60
```

## 4. 网格策略原理

### 核心逻辑

```
1. 计算 N 日最高价 / 最低价
2. 中枢 (mid) = (最高 + 最低) / 2
3. 从中枢向上/下等比展开网格线
   例：11 档、0.5% 步长
       mid × 1.025  ← 最上档
       mid × 1.020
       mid × 1.015
       ...
       mid × 1.000  ← 中间档
       ...
       mid × 0.975  ← 最下档
4. 初始建仓：找到当前价下方的网格线，按比例买入
5. 持续交易：
   - 价格涨穿上档 → 卖出一档
   - 价格跌破下档 → 买入一档
```

### A 股特性

| 特性 | 实现 |
|------|------|
| **T+1** | 当日买入，次日才能卖出 |
| **整手交易** | 100 股为一手，自动取整 |
| **佣金** | 万 2.5，最低 5 元，买卖双向 |
| **印花税** | 千 1，仅卖出收取 |
| **过户费** | 万 0.1，最低 1 元，买卖双向 |

### 仓位计算

```
当前权益 = 现金 + 持仓股数 × 当前股价
目标仓位 = 当前权益 × (档位序号 / 总档位数)
```

例如：11 档网格，当前在第 3 档（从下数），则仓位 = 3/10 = 30%

## 5. 参数说明

| 参数 | CLI 参数 | 默认值 | 范围 | 说明 |
|------|----------|--------|------|------|
| 股票代码 | `--symbol` | 002036 | 6 位代码 | A 股/ETF 代码 |
| 交易所 | `--market` | SZ | SZ/SH | 深圳/上海 |
| 起始日期 | `--start` | 2020-01-01 | YYYY-MM-DD | 回测起点 |
| 结束日期 | `--end` | 2025-12-31 | YYYY-MM-DD | 回测终点 |
| 初始资金 | `--capital` | 100000 | 元 | 起始资金 |
| **回溯周期** | `--period` | 240 | 60-720 | 计算高低点的日数 |
| **网格档位** | `--levels` | 11 | 5-21 | 网格线数量 |
| **每档步长** | `--step` | 0.5 | 0.1-3.0 | 每档百分比间距 |
| MA 闸门 | `--gate` | 关闭 | 开关 | 启用均线过滤 |
| 闸门周期 | `--gate-period` | 60 | 整数 | 均线周期 |

### 参数调优建议

- **大盘股**（茅台、平安）：步长 0.3-0.5%，档位 15-21，周期 240-480
- **中小盘**（联创、东方财富）：步长 0.5-1.0%，档位 11-15，周期 120-240
- **ETF**（沪深300）：步长 0.2-0.5%，档位 11-15，周期 240
- **震荡市**：开启 MA60 闸门（`--gate`），价格在均线下不买入
- **趋势市**：关闭闸门，增大周期和步长

## 6. 数据源管理

### 数据源优先级

```
本地缓存 (.cache/) → baostock → akshare(curl) → efinance
```

### 添加新标的

首次查询时自动下载并缓存。也可手动预缓存：

```bash
python -c "
from src.data_loader import get_stock_data
get_stock_data('600000', 'SH', '2020-01-01', '2025-12-31')
"
```

### 清除缓存

```bash
rm -rf .cache/
# 或只清除特定标的
rm .cache/600519_SH_*.parquet
```

下次运行时会自动重新下载。

### baostock 说明

baostock 是免费开源 A 股数据源，使用原生 TCP socket 协议，**不走 HTTPS**，在受限网络环境下依然可用。数据覆盖沪深全部 A 股（1990 年至今），但不含 ETF 的完整历史数据（ETF 需要 akshare 或 efinance）。

## 7. 常见问题

### Q: 回测结果没有交易？

**原因**：初始资金不足以买入一手。高价股（如茅台 1700 元/股）需要至少 170,000 元才能买一手。

**解决**：提高初始资金 `--capital 200000`，或选低价标的。

### Q: 为什么夏普比率是负数？

网格策略在单边下跌市中会持续买入、越跌越买，导致浮亏扩大。启用 MA60 闸门（`--gate`）可过滤熊市。

### Q: 胜率只有 50% 左右正常吗？

正常。网格策略靠**盈亏比**赚钱，不是靠胜率。每笔盈利（涨一档）> 每笔亏损（跌一档 × 手续费），胜率 50% 即可盈利。

### Q: 端口被占用？

```bash
python server.py 9999     # 换端口
```

### Q: 数据下载失败？

1. 检查网络连接
2. baostock 偶尔超时 → 重试一次
3. akshare 被 CDN 拦截 → 自动启用 curl 补丁
4. 都不行 → 联系管理员更新缓存
