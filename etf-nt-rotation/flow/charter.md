# 项目宪章（charter）

> 立项填。所有 Agent 开工要读的第一份。

- **项目名**：ETF 国家队轮动策略（ETF National-Team Rotation Strategy）
- **目标**（做成什么样算成功，一句话）：构建一套基于国家队持仓变化 + 动量的月度 ETF 轮动策略，回测年化收益 > 15%、夏普 > 1.2、最大回撤 < 20%，并产出可实盘的周度信号推送脚本。
- **范围**：
  - 做：
    - 国家队（汇金、证金、社保基金）持仓数据获取与变化计算
    - ETF 候选池构建（宽基 + 行业 ETF，≥ 30 只）
    - 月度双因子排名（上月涨幅 + 量比）
    - 选股规则：排名第 5-20 名中取中间 5 只，月初等权重买入，月末卖出
    - 国家队仓位变化作为宏观闸门（仓位增减 → 调节总仓位或开关）
    - 完整回测（2019-2026，含手续费和滑点）
    - 实盘信号生成脚本（月底输出下月持仓，PushPlus 推送）
    - 策略绩效归因与敏感性分析
  - 不做：
    - 日内/周频调仓（本策略为月度轮动）
    - 个股选股（仅限 ETF）
    - 期权/期货/杠杆产品
    - 实时交易执行（只到信号生成）
    - Web/GUI 界面
- **约束**（时间/资源/必须遵守的）：
  - 数据源：efinance（东方财富）优先，akshare 回退
  - Python 3.12+，纯脚本无框架依赖
  - 回测代码需与现有 `backtest/` 引擎兼容（可 import 复用）
  - 实盘脚本必须含错误处理和数据完整性校验
- **成功标准**（尽量可衡量）：
  - 回测年化收益率 ≥ 15%
  - 夏普比率 ≥ 1.2
  - 最大回撤 ≤ 20%
  - 月胜率 ≥ 55%
  - 实盘信号脚本可在每月最后一个交易日自动运行并推送
- **角色**：拍板 = 你 / 主控 = Claude Code / 评审 = Codex / 数据 = efinance + akshare
