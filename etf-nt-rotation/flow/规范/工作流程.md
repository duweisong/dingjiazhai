# 工作流程

> 五段式主循环 + 三个核心机制 + 目录归属。这是"活儿怎么流动"的详规。

---

## 五段式主循环(每个任务走一遍)

### ① 立项
填 `flow/charter.md`:目标 / 范围(做什么 + 不做什么)/ 约束 / 成功标准 / 角色。
这是所有 Agent 开工要读的第一份;模糊目标先在这里钉死。

### ② 拆解
出 `flow/plan.md`:里程碑 + 任务拆解(每个任务标:负责角色、输入、产出落哪、验收标准),你确认。
**计划即契约**——要偏离,先改 `plan.md` 再动手。复杂任务先用 Plan 模式出书面计划。

### ③ 执行
每个终端一个角色,读对应任务卡 `flow/tasks/T<编号>-<描述>.md` 开工(文件名带编号+短描述,如 `T01-X-API接入.md`)。
- 可独立的任务并行(git worktree / branch 隔离)。
- 小步、勤检查点。产出**落到 `docs/`(知识)或 `scripts/`·`src/`(代码)**,别只留对话里。

### ④ 评审
交付前**换另一个模型评审**(Codex 审 Claude,或反之),结论写 `docs/` 或任务卡。PASS / FAIL 才算过门。
**铁律:审稿模型 ≠ 产出模型**——错误不相关,等于免费质量倍增。

### ⑤ 沉淀
合并产出;归档完成的任务卡;决策写 `flow/decisions.md`;**问题 / 解决 / 死路 / 踩坑写 `flow/踩坑记录.md`**;从中提炼的禁区 / 心智按 `文档维护SOP.md` 写进 `CLAUDE.md`。

## 核心机制

### A. 进展日志 = 接力棒(Agent 接力的最小单位)
位置:`flow/进展.md`,**append、新的放最上面**。每个 Agent 干完一棒在顶部加一条;**顶部那条 = 当前交接棒**(`CLAUDE.md` 开工必读它)。
每条字段:**做了什么 / 为什么 / 怎么理解 / 产出(路径)/ 问题→解决 / 下一步**。
- 一身三用:**可复制**(整条喂别的 Agent)、**可回看**(append 留全历史,你扫一遍就知道每个 Agent 干了啥)、**能接力**(顶部即最新棒)。
- 收工时这条**既写进文件、也贴在回复里**——用户当场看到 + 可直接复制,不用自己开文件。
- 决策 → `decisions.md`,坑 → `踩坑记录.md`,产物 → `docs/`·`scripts/`(只给路径)。
- **「出方案 → 对照实现」**:A 把方案写 `docs/specs/xxx.md`,进展条的"产出/下一步"带上该路径;B 顺路径打开方案照着实现,再写自己那条(实现中发现的缺口回填进方案)。

**铁律:只带「指针 + 增量」**——指向真实产出(给路径)+ 这一棒新增,**不重抄内容**;接手方要细节顺指针回源文件读。
(为什么是一个 append 文件,而不是 N 张卡或覆盖快照:N 张卡会堆爆;覆盖会丢历史、没法复制;append 一个文件 = 留历史 + 不爆 + 顶部即棒。)

### B. 共用入口 = 运行时合同
`CLAUDE.md` 每个会话自动加载、两个工具都读、是 **Codex 唯一的规则通道**(Codex 没有 skill)。所以**精要规则、约束、目录地图、指针**必须写在这里。`AGENTS.md` 软链到它,写一次两端生效。

### C. 跨模型评审
Claude Code = 主控(持上下文 / 编排);Codex = 独立第二双眼睛。
模式:生成→评审 / 对抗找茬 / 第二意见。何时:每次交付前、高风险逻辑、设计岔路口。

## 目录归属(R1)
| 文件夹 | 放什么 |
|---|---|
| `flow/` | 控制层:charter / plan(契约)/ 进展(append 日志,顶部=交接棒)/ decisions / 踩坑记录 / tasks / 规范副本 |
| `docs/` | 内容层:调研 / `specs/`(方案文档)/ 产出说明 |
| `scripts/` · `src/` | (代码项目)代码产物本身 |
| `CLAUDE.md` | 运行时合同(见机制 B) |
| `DESIGN.md` | (可选)设计 / 创意决策 |

**判据**:这份东西是**协调/推进项目**的 → `flow/`;是**项目要交付的内容** → `docs/`(知识/文档)或 `scripts/`·`src/`(代码)。
控制层**指向**内容层:任务卡 / 进展条里的"产出(路径)"就是 `docs/`·`scripts/` 路径。
