## 策略概述

**一句话**：沪指 K 线 → 15 维市场特征 → Ridge 回归 → **预测**国家队信心指数 → 连续仓位建议(0-100%)；
每月按「涨幅 + 量比」双因子排名，取第 5-20 名中段 5 只 ETF 等权买入，月末清仓。

## 核心逻辑

### 1. 国家队仓位闸门——K线拟合预测模型（宏观层）

**为什么不直接用季度披露数据**：季度数据滞后 1-4 个月，是"后视镜"信号。
**解决方案**：用沪指/HS300 日线 K 线的 15 维市场特征（动量、均线偏离、RSI、波动率、回撤深度等），
通过 Ridge 回归拟合国家队季度信心指数，产出**日频更新的预测仓位**。

**架构**：
```
沪指日线K线 → 15维特征提取 → Ridge回归 → 预测信心指数
    → 三因子公式(信心50%+趋势30%+加速度20%)
    → Sigmoid映射 → 预测仓位 [0.02, 0.98]
    → ETF轮动策略消费（月度调仓时按预测仓位缩放）
```

**仓位映射**（连续值，非离散档位）：
- 预测信心 > +0.20 + 趋势向上 → 80-100%（积极持仓）
- 预测信心 0~0.20 → 60-80%（正常持仓）
- 预测信心 -0.10~0 → 40-60%（中性偏保守）
- 预测信心 -0.20~-0.10 + 趋势向下 → 20-40%（减仓）
- 预测信心 < -0.20 → 0-20%（大幅减仓/空仓）

**数据来源**：
- 预测引擎：`nt-position-sizer/src/predictive_engine.py`（日频K线 → 预测仓位JSON）
- 训练标签：`national_team_tracker.py`（akshare 季度社保持仓 → 训练用标签）
- 回退方案：`nt-position-sizer/src/position_engine.py`（基于最新季度实际数据的仓位）

### 2. 排名因子（选股层）

**因子 1：上月涨幅**
```
momentum = (月末收盘价 - 月初开盘价) / 月初开盘价
```

**因子 2：量比**
```
volume_ratio = 上月日均成交量 / 过去 3 个月日均成交量
```

**综合得分**：
```
score = w1 × momentum_rank + w2 × volume_ratio_rank
# 默认 w1=0.6, w2=0.4（参考 V4.2 经验值）
# 排名升序（得分越低越好，即第 1 名 = 涨幅最大/量比最高）
```

### 3. 选股规则（执行层）

```
1. 计算所有 ETF 的综合得分排名（1 = 最好）
2. 排除排名 1-4（过热，可能追高）
3. 排除排名 21+（动量不足）
4. 从剩余的第 5-20 名（共 16 只）中，选取中间 5 只：
   - 取法：排名第 10, 11, 12, 13, 14（即 5-20 的中段）
   - 备选：排名第 8, 9, 10, 11, 12（偏上中段，可参数化）
5. 等权重分配：每只 20%（总仓位 × 仓位系数）
```

### 4. 调仓节奏

| 时间点 | 动作 |
|--------|------|
| 每月最后一个交易日 | 计算排名 → 生成下月持仓信号 |
| 下月第 1 个交易日 | 开盘买入 5 只目标 ETF |
| 下月最后交易日 | 收盘卖出全部持仓 |
| 循环 | 同时计算再下月信号 |

## 方向决策

- **为什么取 5-20 名中段而非头部**：头部（1-4 名）动量过热，回撤风险高；中段动量确认但不过热，风险收益比更优。尾部（21+）动量不足，缺乏持续性。
- **为什么等权重而非得分加权**：简化执行、降低过拟合风险；得分加权在 V4.2 中已验证有效，但本策略首次迭代先等权再优化。
- **为什么用 K 线预测而非跟随季度披露**：季度数据滞后 1-4 个月，你看到"国家队增持"时他们可能已反向操作；K 线拟合将信号从滞后升级为领先（预判行为而非抄作业）。历史回测（`nt-position-sizer/src/backtest_compare.py`）验证预测仓位策略年化优于实际仓位策略。
